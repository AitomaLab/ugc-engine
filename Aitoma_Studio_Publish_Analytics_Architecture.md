## 1. Architecture Summary

### Services involved

| Service | Runtime | Port (default) | Purpose |
| --- | --- | --- | --- |
| **UGC Backend** (`ugc_backend/main.py`) | FastAPI / Uvicorn | `8000` (alt `8010`) | Auth, social scheduling, Ayrshare passthrough, analytics router (mounted), Stripe, content jobs |
| **Creative OS** (`services/creative-os/main.py`) | FastAPI / Uvicorn | `8001` (alt `8011`) | Multi-asset campaign planner, campaign watcher loop, agent endpoints, Wavespeed/KIE webhooks |
| **Frontend** (`frontend/`) | Next.js 16 (Turbopack) | `3000` (alt `3010`) | Publish UI (Calendar + Analytics tabs), schedule modal, Create wizard, dark dashboard |
| **Celery worker** (optional) | Celery + Redis | n/a | Long-running video jobs; degrades to in-process daemon thread when Redis unreachable |
| **Modal serverless** (optional) | Modal webhook | external | Alternative video-job dispatcher, gated by `USE_MODAL_WORKER=true` |
| **Supabase** | Managed Postgres + Storage + Auth | external | Single source of truth for all relational data + bucket `analytics-media` for mirrored video/poster files |
| **Redis** | local or Upstash | `6379` | Celery broker; falls back gracefully when not present |

### High-level data flow

1. **Generate** — User creates a video / image asset (`/create` flow), producing a row in `video_jobs` or `product_shots` with a final media URL on Supabase Storage.
2. **Schedule** — User opens **Publish → Calendar** (`frontend/src/app/schedule/CalendarTab.tsx`), drops the asset on a date via `SchedulePostModal`. Frontend posts to `POST /api/schedule/bulk` which:
   - Inserts one `social_posts` row per (asset, platform).
   - Calls Ayrshare `POST /api/post` with the same caption + `scheduleDate` (UTC ISO 8601).
   - Stores the Ayrshare-side ID on the row.
   - Fires a debounced `BackgroundTasks` analytics sync (`analytics_studio_service.sync_studio_connections_for_user`).
3. **Publish (Ayrshare)** — Ayrshare publishes to TikTok / Instagram / YouTube / Facebook on its own infrastructure. When publishing succeeds or fails, it fires a webhook to `POST /api/webhooks/ayrshare` (HMAC-signed with `AYRSHARE_WEBHOOK_SECRET`) and the backend flips the `social_posts.status` to `posted` / `failed`.
4. **Sync** — `studio_service.sync_studio_connections_for_user(user_id)` mirrors:
   - OAuth handles from `GET /api/user` (Ayrshare) → `analytics_tracked_accounts` (`linked_via_connections=true`).
   - `social_posts` rows where `status='posted'` → `analytics_posts` (`source='internal'`).
   - Live engagement from `POST /api/analytics/post` (Ayrshare) → `analytics_posts.{views,likes,comments,shares,saves,impressions,reach,clicks,ctr,media_type}`.
   - One `analytics_video_breakdowns` row per Studio video with `status='pending'`, then dispatches to a daemon thread.
5. **External tracking** — User pastes a URL or `@handle` into `AnalyzeSearchBar`. Backend triggers `BrightData Datasets v3` via `scraper_service.scrape()`, polls the snapshot, normalizes records into `analytics_posts` (`source='external'`), and mirrors the video + poster frame into the `analytics-media` Supabase bucket via a daemon thread.
6. **AI breakdown** — `vision_service.analyze_video()` runs a two-pass Gemini call (Pass 1: structured JSON of hook/scenes/audio/visual_details/key_moments, Pass 2: textual takeaways using the post's metrics). Result is persisted on `analytics_video_breakdowns`.
7. **Surface** — Frontend dashboard (`AnalyticsTab` → `DashboardView`) calls `/api/analytics/stats`, `/stats/distribution`, `/stats/cumulative`, and per-account endpoints. KPI cards, growth chart, and distribution panels render from a single composed payload per period.
8. **Feedback into agent** — Creative OS reads `agent_memories` (per-user persistent notes) on every agent turn. The Campaign Watcher consumes `campaign_plan_items` and emits `social_posts` rows the same way the manual scheduler does, so dashboard metrics surface for both automated and manual campaigns identically.

### Connection to the rest of Aitoma Studio

- **Auth**: All routes use `ugc_backend.auth.get_current_user`, which validates a Supabase Auth JWT against `SUPABASE_URL` + `SUPABASE_ANON_KEY`. RLS is the second line of defense — every analytics / publish table has `ENABLE ROW LEVEL SECURITY` and an `auth.uid() = user_id` policy.
- **Storage**: Two buckets are used — `analytics-media` (mirrored external videos + poster frames) and `product-images` (compressed Instagram-ready uploads from `schedule_media.py`).
- **Redis / Celery**: Publish + analytics paths do **not** require Redis. Analytics jobs explicitly use in-process daemon threads (`analytics/jobs.py:run_breakdown_in_background`). Video-generation jobs use Celery when `redis://localhost:6379/0` is reachable, else in-process.
- **Webhooks**:
  - `POST /api/webhooks/ayrshare` — post status & social-account changes (HMAC-validated).
  - `POST /api/stripe/webhook` — credit top-ups (signed via `STRIPE_WEBHOOK_SECRET`); not analytics-related but lives on the same FastAPI app.
  - `POST /creative-os/webhooks/wavespeed` — image generation callbacks (Creative OS service).

### Deployment surface

| Component | Where it runs |
| --- | --- |
| `ugc_backend` (publish + analytics + auth) | Single Uvicorn process. Locally launched via `python app.py`. Production deploys to Railway / Fly / Render — single image with no Redis dependency. |
| `services/creative-os` | Sibling Uvicorn process (`uvicorn main:app --port 8001`). Hosts the campaign watcher (`asyncio` background task) so it must run as a long-lived process; serverless is not viable. |
| `frontend` | Next.js on Vercel (or `pnpm dev` locally on `:3010`). Talks to `NEXT_PUBLIC_API_URL` and `NEXT_PUBLIC_CREATIVE_OS_URL` (or a same-origin rewrite). |
| Postgres | Supabase managed. All migrations live in `ugc_db/migrations/*.sql`. |
| Redis | Optional. Local or Upstash. Disabling it does **not** affect publish/analytics — only video jobs degrade to in-process. |
| Ayrshare | Multi-tenant SaaS. Each Aitoma user has a sub-profile (`ayrshare_profiles.ayrshare_profile_key`). |
| BrightData | Datasets v3 — one dataset per (platform, kind). Polled synchronously from inside `scraper_service.scrape()`. |
| Gemini (Google AI) | Direct via `google-genai` (preferred) or proxied through FAL / KIE. |

## 2. Database Schema

All publish / analytics tables live in the `public` schema, are RLS-locked to `auth.uid() = user_id`, and ship via numbered SQL migrations under `ugc_db/migrations/`.

### 2.1 Publishing & scheduling

```sql
-- Migration 012: Ayrshare profile mapping (one row per user).
CREATE TABLE ayrshare_profiles (
    user_id              UUID PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
    ayrshare_profile_key TEXT NOT NULL UNIQUE,
    connected_accounts   JSONB DEFAULT '{}',
    created_at           TIMESTAMPTZ DEFAULT NOW(),
    updated_at           TIMESTAMPTZ DEFAULT NOW()
);

ALTER TABLE ayrshare_profiles ENABLE ROW LEVEL SECURITY;
CREATE POLICY ayrshare_profiles_user_policy ON ayrshare_profiles
    FOR ALL USING (user_id = auth.uid());

-- Migration 012 + 037: persistent record of every scheduled / posted post.
CREATE TABLE social_posts (
    id                UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id           UUID REFERENCES auth.users(id) ON DELETE CASCADE,
    video_job_id      UUID REFERENCES video_jobs(id) ON DELETE CASCADE,        -- nullable since 037
    product_shot_id   UUID REFERENCES product_shots(id) ON DELETE SET NULL,    -- added in 037
    media_kind        TEXT NOT NULL DEFAULT 'video'
                      CHECK (media_kind IN ('video','image')),                  -- 037
    ayrshare_post_id  TEXT,
    status            TEXT NOT NULL DEFAULT 'scheduled',
    platform          TEXT NOT NULL,
    caption           TEXT,
    hashtags          TEXT[],
    scheduled_at      TIMESTAMPTZ NOT NULL,
    posted_at         TIMESTAMPTZ,
    error_message     TEXT,
    created_at        TIMESTAMPTZ DEFAULT NOW(),
    updated_at        TIMESTAMPTZ DEFAULT NOW(),
    -- Exactly one media FK populated (XOR enforced).
    CONSTRAINT chk_social_posts_media_source CHECK (
        (media_kind = 'video' AND video_job_id IS NOT NULL AND product_shot_id IS NULL)
     OR (media_kind = 'image' AND product_shot_id IS NOT NULL AND video_job_id IS NULL)
    )
);

ALTER TABLE social_posts ENABLE ROW LEVEL SECURITY;
CREATE POLICY social_posts_user_policy ON social_posts
    FOR ALL USING (user_id = auth.uid());

CREATE INDEX idx_social_posts_user_id        ON social_posts (user_id);
CREATE INDEX idx_social_posts_status         ON social_posts (status);
CREATE INDEX idx_social_posts_scheduled_at   ON social_posts (scheduled_at);
CREATE INDEX idx_social_posts_product_shot_id ON social_posts (product_shot_id);
```

```sql
    linked_via_connections  BOOLEAN NOT NULL DEFAULT FALSE,                           -- 036

CREATE INDEX idx_ata_user                 ON analytics_tracked_accounts (user_id);

CREATE POLICY ata_user_policy ON analytics_tracked_accounts
    FOR ALL USING (user_id = auth.uid());

CREATE TABLE analytics_posts (
    id                  UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id             UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
    source              TEXT NOT NULL CHECK (source IN ('internal','external')),
    social_post_id      UUID REFERENCES social_posts(id) ON DELETE SET NULL,
    video_job_id        UUID REFERENCES video_jobs(id)  ON DELETE SET NULL,
    platform            TEXT NOT NULL,
    username            TEXT NOT NULL,
    post_url            TEXT NOT NULL,
    external_post_id    TEXT,
    caption             TEXT,
    hashtags            TEXT[],
    media_type          TEXT,
    media_urls          JSONB DEFAULT '[]'::jsonb,
    storage_video_url   TEXT,                                       -- 034
    thumbnail_url       TEXT,
    duration_seconds    NUMERIC,
    posted_at           TIMESTAMPTZ,
    added_at            TIMESTAMPTZ DEFAULT NOW(),                  -- 034 (period filter)
    -- Engagement
    views               BIGINT,
    likes               BIGINT,
    comments            BIGINT,
    shares              BIGINT,
    saves               BIGINT,
    -- Funnel (039)
    impressions         BIGINT,
    reach               BIGINT,
    clicks              BIGINT,
    ctr                 NUMERIC(6,4),                               -- 0.0000-1.0000
    -- Generated columns (never written from app code)
    total_engagement    BIGINT GENERATED ALWAYS AS (
        COALESCE(likes,0) + COALESCE(comments,0)
      + COALESCE(shares,0) + COALESCE(saves,0)
    ) STORED,
    is_studio_published BOOLEAN GENERATED ALWAYS AS
        (social_post_id IS NOT NULL) STORED,                        -- 035
    raw_payload         JSONB,
    scraped_at          TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE (user_id, platform, post_url)
);

ALTER TABLE analytics_posts ENABLE ROW LEVEL SECURITY;
CREATE POLICY ap_user_policy ON analytics_posts
    FOR ALL USING (user_id = auth.uid());

CREATE INDEX idx_analytics_posts_user                  ON analytics_posts (user_id);
CREATE INDEX idx_analytics_posts_posted_at             ON analytics_posts (posted_at DESC);
CREATE INDEX idx_analytics_posts_added_at              ON analytics_posts (added_at DESC);   -- 034
CREATE INDEX idx_analytics_posts_engagement            ON analytics_posts (total_engagement DESC);
CREATE INDEX idx_analytics_posts_platform              ON analytics_posts (platform);
CREATE INDEX idx_analytics_posts_source                ON analytics_posts (source);
CREATE INDEX idx_analytics_posts_is_studio_published   ON analytics_posts (is_studio_published);
```

### 2.5 Campaigns (Creative OS planner — feeds Publish via watcher)

```sql
-- Migration 027: durable multi-asset content campaigns.
CREATE TABLE campaigns (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id         UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
    project_id      UUID REFERENCES projects(id) ON DELETE SET NULL,

    goal            TEXT,
    branding_notes  JSONB NOT NULL DEFAULT '{}'::jsonb,
    start_date      DATE,
    end_date        DATE,
    cadence         JSONB NOT NULL DEFAULT '{"interval":"daily","time_utc":"15:00"}'::jsonb,

                    CHECK (status IN ('planning','approved','running','completed','failed','cancelled')),
    plan_json       JSONB,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

    id                  UUID PRIMARY KEY DEFAULT gen_random_uuid(),

    scheduled_at        TIMESTAMPTZ NOT NULL,
    asset_type          TEXT NOT NULL
                        CHECK (asset_type IN ('ugc_video','clone_video','product_shot','generated_image','animated_image')),
    brief               JSONB NOT NULL DEFAULT '{}'::jsonb,
    platforms           TEXT[] NOT NULL DEFAULT '{}',
    caption             TEXT,
    hashtags            TEXT[] DEFAULT '{}',
    job_id              UUID,                          -- video_jobs.id once dispatched
    asset_url           TEXT,                          -- final media URL
    scheduled_post_id   UUID,                          -- social_posts.id once Ayrshare schedules
    status              TEXT NOT NULL DEFAULT 'pending'
                        CHECK (status IN ('pending','generating','ready_to_post','scheduled','posted','failed','cancelled')),
    error               TEXT,
    created_at          TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at          TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    UNIQUE (campaign_id, slot_index)
);

CREATE INDEX idx_campaign_items_campaign_status ON campaign_plan_items (campaign_id, status);
CREATE INDEX idx_campaign_items_job             ON campaign_plan_items (job_id) WHERE job_id IS NOT NULL;
CREATE INDEX idx_campaign_items_watcher         ON campaign_plan_items (status, updated_at)
    WHERE status IN ('generating','ready_to_post');

ALTER TABLE campaigns           ENABLE ROW LEVEL SECURITY;
ALTER TABLE campaign_plan_items ENABLE ROW LEVEL SECURITY;

CREATE POLICY campaigns_owner ON campaigns
    FOR ALL TO authenticated USING (user_id = auth.uid()) WITH CHECK (user_id = auth.uid());
CREATE POLICY campaign_items_owner ON campaign_plan_items
    FOR ALL TO authenticated
    USING      (EXISTS (SELECT 1 FROM campaigns c WHERE c.id = campaign_id AND c.user_id = auth.uid()))
    WITH CHECK (EXISTS (SELECT 1 FROM campaigns c WHERE c.id = campaign_id AND c.user_id = auth.uid()));
```

## 4. External Integrations

### 4.1 Ayrshare (publish + Studio metrics)

- **Base URL**: `https://app.ayrshare.com/api`
- **Auth**: `Authorization: Bearer ${AYRSHARE_API_KEY}` plus per-user `Profile-Key: ${ayrshare_profile_key}`.
- **Client**: `ugc_backend/ayrshare_client.py` (httpx async).

| Operation | Endpoint | Where |
| --- | --- | --- |
| Create user sub-profile | `POST /profiles/profile` | `_provision_ayrshare_profile` (main.py) |
| Generate JWT (link popup) | `POST /profiles/generateJWT` | `POST /api/ayrshare/jwt` |
| List connected socials | `GET /user` (fallback `GET /profiles`) | `GET /api/connections` |
| Schedule / publish post | `POST /post` | `POST /api/schedule/bulk`, campaign watcher |
| Cancel scheduled post | `DELETE /post` | `DELETE /api/schedule/{id}` |
| Per-post analytics | `POST /analytics/post` | `studio_service.refresh_studio_post_metrics` |

**Request shape (publish)**:

```json
{
  "post": "Caption text",
  "platforms": ["instagram"],
  "mediaUrls": ["https://…/video.mp4"],
  "scheduleDate": "2026-06-01T15:00:00Z",
  "hashTags": ["ai", "ugc"]
}
```

**Response (success)**: `{ "id": "ayr_xxx", "status": "success", "posts": [...] }`

**Errors**:
- HTTP `>= 400` → `_friendly_ayrshare_post_error()` extracts the nested `posts[*].errors[*].message` envelope and raises `ValueError`.
- Code `144` ("Invalid Profile Key") raises `InvalidProfileKey`. The route handlers catch it, drop the `ayrshare_profiles` row, and re-provision a fresh sub-profile transparently.

**Polling / retries**: No outbound polling. State changes arrive via webhook.

**Inbound webhook** — `POST /api/webhooks/ayrshare` (`ugc_backend/main.py:3433`):

```python
# HMAC SHA-256 verification.
expected = request.headers.get("X-Authorization-Content-SHA256")
computed = hmac.new(AYRSHARE_WEBHOOK_SECRET.encode(), body, hashlib.sha256).hexdigest()
if not hmac.compare_digest(computed, expected): raise 401
```

- `action == "post"`: flips `social_posts.status` to `posted` (with `posted_at`) or `failed` (with `error_message`).
- `action == "social"`: mirrors `activeSocialAccounts` into `ayrshare_profiles.connected_accounts`.

### 4.2 BrightData Datasets v3 (external scraping)

- **Base URL**: `https://api.brightdata.com`
- **Auth**: `Authorization: Bearer ${BRIGHTDATA_API_KEY}`
- **Client**: `ugc_backend/analytics/scraper_service.py`
- **Datasets**: one ID per (platform × kind), e.g. `BRIGHTDATA_TIKTOK_POST_DATASET_ID`, `BRIGHTDATA_INSTAGRAM_PROFILE_DATASET_ID`. 8 supported pairs (TikTok / IG / YouTube / Facebook × post / account).

| Step | Endpoint |
| --- | --- |
| Trigger collection | `POST /datasets/v3/trigger?dataset_id=…&format=json&include_errors=true` body `[{"url": "<canonical url>"}]` |
| Poll snapshot | `GET /datasets/v3/snapshot/{snapshot_id}?format=json` |

**Polling logic** (`_poll_snapshot`):
- Initial interval `BRIGHTDATA_POLL_INTERVAL=4s`.
- Exponential backoff `interval = min(interval * 1.5, 15s)`.
- Hard ceiling `BRIGHTDATA_MAX_WAIT=120s` — beyond that, the scrape returns `status="pending"` and the FE keeps polling `/api/analytics/scrape-jobs`.
- `HTTP 202` → still running. `HTTP 200 with body status='ready'` → records.

**Cost tracking**: `_PER_RECORD_COST_USD = 0.002` (overridable). `analytics_scrape_jobs.estimated_cost_usd` = `records_returned × per_record_cost`.

**Mock mode**: `BRIGHTDATA_MOCK=true` short-circuits to `analytics/fixtures/{platform}_post.sample.json`.

**Error handling**:
- Trigger HTTP `>= 400` → `RuntimeError` with response body excerpt; `analytics_scrape_jobs.status='failed'`.
- Per-record `{ "error_code": "…", "error": "…" }` envelopes → if EVERY record is an error envelope, the whole scrape is marked `failed` with the upstream message; otherwise individual error rows are dropped silently.
- Snapshot `status='failed'` → `RuntimeError` propagated to caller.
- All BrightData media URLs are mirrored to `analytics-media` Supabase bucket (`_mirror_video_to_storage`) because they are short-lived & CORS-blocked. ffmpeg extracts a poster frame at `t=1s`.

### 4.3 Gemini (AI breakdown — `vision_service.py`)

Two-pass pipeline. The pipeline picks **separate** providers for the upload-heavy pass 1 and the text-only pass 2.

| Pass | Inputs | Provider order | Output |
| --- | --- | --- | --- |
| Pass 1 | Video file (downloaded to temp) + prompt | `GEMINI_API_KEY` (google-genai Files API) → `FAL_KEY` (FAL Gemini app) | Strict JSON: `{summary, hook, scenes[], audio, visual_details, key_moments[]}` |
| Pass 2 | Pass 1 JSON + post metrics (views/likes/…) | `KIE_API_KEY` (OpenAI-compatible chat) → `FAL_KEY` → `GEMINI_API_KEY` | `takeaways[]` (3 strategic items) |

KIE is **excluded from Pass 1** because its OpenAI-compatible `/v1/files` endpoint cannot accept the Gemini video MIME types.

**Retry**: Pass 1 retries once on invalid JSON (parser bails on the JSON-extraction regex). Failures sanitised through `_friendly_error()` and surfaced on `analytics_video_breakdowns.error_message`.

**Concurrency**: Each breakdown runs in a daemon thread (`analytics/jobs.py:run_breakdown_in_background`). Status transitions: `pending → running → completed | failed`. Idempotent — re-running on the same `breakdown_id` overwrites the row.

### 4.4 Social platform APIs (indirect)

The publish pipeline never touches TikTok / IG / YouTube / Facebook directly — Ayrshare brokers OAuth + posting. The analytics pipeline never authenticates with platforms either; it scrapes via BrightData (public data). Ayrshare's `/analytics/post` is the only authenticated read of platform-side metrics.

### 4.5 Stripe (orthogonal but on the same FastAPI app)

- `POST /api/stripe/webhook` — verifies `STRIPE_WEBHOOK_SECRET`, credits the user via `add_credits()`. Not part of the analytics pipeline; documented for completeness because it shares the same webhook surface.

## 6. Analytics Queries & Feedback Loop

### 6.1 Aggregations powering the dashboard

The `/api/analytics/stats` endpoint **composes** its response from three pure-data helpers in `analytics/db.py`. The original `stats()` contract was preserved (guardrail) — new helpers were added alongside.

```python
# ugc_backend/analytics/router.py — composed dashboard payload.
@router.get("/stats", response_model=StatsResponse)
def api_stats(period: Optional[str] = Query("30d"), user = Depends(get_current_user)):
    days   = _period_days(period)                                # 7 / 30 / 90 / None
    base   = analytics_db.stats(user["id"], days=days)            # totals only (legacy contract)
    extras = analytics_db.stats_extras(user["id"], days=days)     # daily sparklines + deltas
    dist   = analytics_db.stats_distribution(user["id"], days=days)  # platform + media buckets
    return StatsResponse(**base, **extras, **dist)
```

| Metric | Source query (conceptual) |
| --- | --- |
| `total_views` | `SELECT SUM(views) FROM analytics_posts WHERE user_id=$1 AND added_at >= now() - interval '${days} days'` |
| `total_engagement` | Same window, `SUM(total_engagement)` (generated column). |
| `avg_engagement_rate` | `SUM(total_engagement)::numeric / NULLIF(SUM(views), 0)` over same window. |
| `posts_tracked` | `COUNT(*)` over same window. |
| Sparkline (`stats_extras`) | `SELECT date_trunc('day', added_at) AS d, SUM(views), SUM(total_engagement), COUNT(*) FROM analytics_posts WHERE user_id=$1 AND added_at BETWEEN now() - interval '${days*2} days' AND now() GROUP BY 1 ORDER BY 1` — first half = previous period (delta), second half = sparkline. |
| Distribution (`stats_distribution`) | `SELECT platform, SUM(views), COUNT(*) FROM analytics_posts WHERE user_id=$1 AND added_at >= … GROUP BY platform` + same by `media_type`. |
| Cumulative (`stats_cumulative` → `/stats/cumulative`) | Daily totals seeded by pre-period data so the curve starts at the running total at `T-${days}`, not at 0. |
| Account aggregates (`/api/analytics/accounts`) | LEFT JOIN `analytics_tracked_accounts` ↔ `analytics_posts` per `(platform, username)`; aggregates `total_views`, `total_engagement`, `top_post_url`, `last_post_at`. |
| Top posts per account (`/accounts/{id}/top-posts`) | `ORDER BY total_engagement DESC LIMIT N`. |
| Trend per account (`/accounts/{id}/trend`) | Daily `SUM(views)` + `SUM(total_engagement)` for that `(platform, username)`. |
| CSV export (`/export/csv`) | All `analytics_posts` rows with the same period filter, streamed via `StreamingResponse`. |

### 6.2 Scheduled / event-driven jobs

There is **no global cron** — every refresh is event-driven. Triggers:

| Trigger | Action |
| --- | --- |
| `GET /api/connections` (page load) | Calls `analytics_studio_service.claim_debounced_sync(user_id)` (15s lock). If unlocked, schedules `BackgroundTasks(sync_studio_connections_for_user)`. |
| `POST /api/schedule/bulk` (after success) | Calls `allow_immediate_sync(user_id)` (clears debounce) + schedules the same background sync so the new post lands in `analytics_posts` immediately. |
| `POST /api/analytics/sync-studio-connections` | Manual full sync invoked from the dashboard refresh button. |
| `POST /api/analytics/refresh-all` | Runs Studio metrics refresh (Ayrshare) + external metrics refresh (BrightData per-post) + queues breakdowns. |
| `POST /api/analytics/posts/refresh` | Per-post live refresh from Ayrshare or BrightData. |
| `POST /api/analytics/scrape` | One-shot BrightData run (URL or `@handle`). |
| `POST /api/analytics/tracked-accounts/{id}/refresh` | Per-account scrape, capped at `top_n_retention`. |
| Ayrshare webhook `action=post` | Flips `social_posts.status` to `posted` / `failed`. The Studio sync (next page load) picks up the new metrics. |
| Creative OS Campaign Watcher (`workers/campaign_watcher.py`) | Every `CAMPAIGN_WATCHER_INTERVAL=30s`, advances `campaign_plan_items`. When a plan-item lands at `scheduled`, the resulting `social_posts` row flows through the same analytics pipeline as a manually-scheduled post. |

**Stale-metrics guard**: `studio_service.metrics_refresh_is_stale()` checks `analytics_settings.last_metrics_refreshed_at`. If older than `METRICS_REFRESH_INTERVAL_HOURS = 24`, the next sync re-fetches Ayrshare metrics; otherwise it short-circuits. `force_metrics=True` overrides on explicit user refresh.

### 6.3 Feedback into the agent's creative decisions

The feedback path is intentionally **explicit + user-mediated** today, with two surfaces:

1. **"Use as Template" CTA** (`PostDetailModal.tsx`) — the user clicks the green **Use as Template** button on a high-performing post. The modal builds a URL like:

   ```
   /create
     ?customScript=<caption + AI hook + takeaways>
     &templatePostId=<analytics_posts.id>
     &templatePlatform=<platform>
     &templateSource=<internal|external>
   ```

   `app/create/page.tsx` reads `customScript`, pre-fills the script textarea, and sets `scriptSource='custom'`. This converts a winning competitor post into an Aitoma generation brief without mutating any DB row.

2. **Persistent agent memory** (`agent_memories` table, migration `028`) — the creative-director agent reads a memory snapshot on every first turn (`messages_agent_client.py:214`):

   ```python
   snapshot = await _agent_memory.read_snapshot(user_token, user_id)
   prefix   = "[Memory snapshot — your persistent notes about this user. ...]"
   first_turn_messages.insert(0, {"role": "user", "content": prefix + snapshot})
   ```

   The agent has a `memory` tool that can `read` / `write` paths under `/memories/…`. This is where the agent stores brand voice, hooks that worked, and other durable facts. **Today this is populated by user-correction during chat; the auto-mining pipeline (analytics → agent_memories) is a planned extension — the storage shape is already in place.**

3. **Campaign re-planning** — the watcher (`workers/campaign_watcher.py`) is status-driven, not metric-driven. It does not currently consume `analytics_posts` to re-rank assets. When auto-feedback is wired up, the natural integration point is `services/campaign_planner.py` (system prompt is a single string at module top-level) which can be extended to read top performers per platform from `analytics_posts` before generating the next plan.

### 8.8 Required vs optional matrix

| Variable | Required for | Notes |
| --- | --- | --- |
| `SUPABASE_URL`, `SUPABASE_SERVICE_KEY` | All publish + analytics | Service-role for backend writes; never exposed to browser. |
| `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Frontend session | RLS-scoped reads. |
| `AYRSHARE_API_KEY`, `AYRSHARE_DOMAIN`, `AYRSHARE_PRIVATE_KEY*` | Publish flow + `/api/connections` | Without these, scheduling fails fast with 400. |
| `AYRSHARE_WEBHOOK_SECRET` | Optional | If unset, webhooks skip HMAC validation (NOT recommended for prod). |
| `BRIGHTDATA_API_KEY` + dataset IDs | External account/post tracking | `BRIGHTDATA_MOCK=true` bypasses all of these for local dev. |
| `GEMINI_API_KEY` **or** `FAL_KEY` | AI breakdown Pass 1 | At least one of these must be set or breakdowns return `failed`. |
| `KIE_API_KEY` / `FAL_KEY` / `GEMINI_API_KEY` | AI breakdown Pass 2 | Any one suffices. |
| `OPENAI_API_KEY` | Caption suggestions (`/api/schedule/generate-caption`) | Optional; falls back to canned captions on failure. |
| `CELERY_BROKER_URL`, `REDIS_URL` | Optional | Video jobs degrade to in-process. Analytics never depends on Redis. |
| `STRIPE_*` | Billing only | Not part of the publish/analytics pipeline. |
